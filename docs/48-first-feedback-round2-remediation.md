# 48-首轮反馈剩余问题修整文档

## 1. 背景
上一轮已完成：
- 导航结构改为一级分组折叠
- 移动端适配基础
- 文案部分收敛
- 上传来源入口
- 注册入口闭环

但用户本轮反馈里仍有若干剩余问题未收口，需要单独修整：
1. 注册时用户名/邮箱/密码不符合要求时，前端没有友好提示，而是直接 401/失败提示
2. 页面顶栏标题样式与文案不符合要求，应改为 `笔记 / 笔记生成` 这种单行形式，并沿用“笔记”样式
3. 普通用户不应在设置模块看到“用户与登录情况”
4. 页面多余/过时文字仍偏多，需要继续调查并收敛
5. 用户活动概览数据全 0，需要排查是程序问题还是数据链路问题，并修复
6. 视觉风格需要继续对齐“织物质感”要求，避免新加页面/区块风格跑偏

## 2. 目标
对上述剩余问题做第二轮定向修整，并完成回归验证。

## 3. 子任务拆分
| SA | 目标 | 类型 |
|---|---|---|
| SA-20D | 前端剩余体验修整（注册校验、顶栏标题、普通用户隐藏用户与登录情况、文案继续收敛、织物风格对齐） | 实现 |
| SA-20E | 用户活动概览为 0 的链路排查与修复 | 实现 |
| SA-20F | 第二轮剩余问题回归验证 | 验证 |

## 4. 验收标准
- 注册表单在前端给出明确字段级提示
- 顶栏标题改为单行路径式展示
- viewer 不显示“用户与登录情况”设置项
- 解释性/过时文案继续减少
- 活动概览有真实数据或能明确复现与解释为 0 的原因
- UI 细节更贴合织物质感要求
