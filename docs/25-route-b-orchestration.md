# 25-路线B实施编排文档

## 1. 当前阶段目标
在路线 A 完成后，正式进入路线 B（体验优化 / 真功能增强），优先做最影响真实使用体验的增强项。

## 2. 子任务拆分
| SA | 目标 | 类型 | 依赖 |
|---|---|---|---|
| SA-12A | 异步任务真实化与任务体验增强 | 实现 | 路线A基线 |
| SA-12B | PDF / 多模态处理质量增强 | 实现 | 现有 notes 主链 |
| SA-12C | 知识点抽取与用户活动埋点增强 | 实现 | 现有 review/settings 主链 |

| SA-12E | 路线B阶段验收 | 验证 | SA-12A/B/C/D |

## 3. 优先级
优先顺序：
1. SA-12A 异步任务真实化
2. SA-12B PDF / 多模态增强
3. SA-12C 知识点抽取与埋点增强
4. SA-12D UI/交互优化
5. SA-12E 阶段验收

## 4. 质量门禁
- 不破坏路线 A 已达成的构建/部署基线
- 每条增强链路都需有可验证证据
- 路线 B 结束前必须再次做整体验证
