# 06-实现总方案文档

## 1. 实现策略
采用“骨架先行、主链打通、再补充外围能力”的方式推进：

1. 后端先完成认证/配置/基础模型/任务机制。
2. 再完成资料扫描与笔记生成链路。
3. 再完成 FSRS 复习链路、总结与思维导图生成。
4. 前端并行落地壳层与各模块页面。
5. 最后补齐 Docker、导入导出、最终验证。

## 2. MVP 主链
- 管理员登录
- 配置 AI 与工作目录
- 扫描文件夹并登记来源文件
- 选择资产触发笔记生成
- 查看 Markdown 笔记详情
- 生成知识点 / 初始化复习卡
- 普通用户登录并复习
- 生成知识点总结 / 思维导图
- 管理员触发 Obsidian 同步与数据库导出

## 3. 后端实现方式
- FastAPI 分模块 router
- SQLAlchemy 2.0 异步模式
- Celery worker 处理重任务
- Provider Adapter 适配 OpenAI 兼容 API
- 文件服务负责安全路径映射与写入
- 审计服务记录登录/查看/复习行为

## 4. 前端实现方式
- React Router 组织三级导航
- 统一 AppShell 左侧边栏
- 权限态按钮统一组件处理 disable + tooltip
- 织物质感主题由 Tailwind tokens + 背景纹理 + 阴影系统实现
- 编辑/查看分离，首版以查看与触发任务为主

## 5. Obsidian 同步实现方案
### 5.1 首选模式：本地 Vault 目录直写
- 用户将工作目录指向 Obsidian Vault 或其子目录。
- 系统直接生成/更新 Markdown 与附件。

### 5.2 增强模式（当前已批准为首版实现目标）
- 管理员配置 `obsidian-headless` 可执行路径、vault 名称/ID、config dir、device name。
- 后端在写入后可触发 `ob sync` 或相关 setup/sync 命令。
- 同步命令作为增强能力纳入首版实现，而不是仅预留。

### 5.3 扩展模式
- 预留自定义同步 hook：支持 shell command / webhook / LiveSync 外部脚本。

## 6. AI 适配方案
- LLM/Embedding/STT/OCR 分别独立 provider 配置。
- 默认按 OpenAI 兼容请求格式发送。
- OCR 可用多模态 LLM，也可单独 OCR 模型。
- STT 支持远程 OpenAI 兼容接口，也支持本地兼容服务。
- PDF 首版直接按多模态输入源处理，不单做传统 OCR+版面重建链。

## 7. 数据导入导出
- 导出：数据库快照 + 配置脱敏 JSON + 生成产物索引
- 导入：管理员上传导出包后执行恢复/合并

## 8. 安全控制
- 工作目录必须在管理员允许根目录下
- 禁止路径穿越
- 普通用户无写文件能力
- 管理员可查看用户行为报表
