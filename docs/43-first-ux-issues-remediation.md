# 43-人工测试前首轮体验问题修整文档

## 1. 用户反馈
在尚未进入笔记生成流程前，用户已发现以下问题：
1. 移动端页面不友好
2. 页面解释性文字过多
3. 侧边栏不应分成两层，应改为一级分组（笔记/复习/设置）可展开收起子项
4. 笔记生成页需要增加“直接网页上传来源”功能，且作为首选，而不是手动扫描目录
5. 开启新用户注册后仍没有注册入口

## 2. 目标
在不破坏现有主链的前提下，先完成这批首轮体验问题的修整。

## 3. 拆分原则
- 先做前端信息架构 / 交互体验问题
- 再做需要后端支撑的上传功能
- 最后做回归验证

## 4. 子任务拆分
| SA | 目标 | 类型 |
|---|---|---|
| SA-20A | 前端导航/移动端/文案收敛/注册入口修整 | 实现 |
| SA-20B | 笔记生成页网页上传功能（含后端最小支撑） | 实现 |
| SA-20C | 首轮体验问题回归验证 | 验证 |

## 5. 验收标准
- 移动端可基本使用
- 侧边栏改为一级可折叠分组结构
- 解释性文字明显收敛
- 上传入口可用且成为首选路径
- 开启注册后，登录页存在注册入口
