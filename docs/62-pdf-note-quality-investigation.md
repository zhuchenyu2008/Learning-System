# 62-PDF笔记生成质量问题排查文档

## 1. 用户反馈
用户在最新 Docker 环境中仅测试了 PDF 一种来源，发现生成出来的笔记质量明显异常，说明虽然“来源 -> 文本 -> 笔记”的主链结构已重构，但 PDF 这条来源链在真实效果上仍有问题。

## 2. 当前目标
针对 PDF 来源单独排查：
1. PDF 实际进入生成阶段前，提取出的文本到底是什么
2. 规范化文本是否失真、过短、脏数据过多或结构错误
3. 最终 Markdown 笔记异常是提取问题、规范化问题、Prompt 问题，还是写回模板问题

## 3. 排查范围
- `backend/app/services/note_generation_service.py`
- `backend/app/integrations/pdf_processing.py`
- 与 PDF OCR / 多模态提取 / 文本规范化 / prompt 拼装相关的最小范围
- 真实 Docker 环境中的 PDF 上传 -> 生成 -> job/log/note 文件核验

## 4. 不要做
- 不扩展新功能
- 不顺手大改其他来源类型
- 不把“链路能跑通”和“生成质量合格”混为一谈

## 5. 验收标准
- 明确拿到一份真实 PDF 生成链的中间证据：提取文本、规范化文本、最终生成结果
- 定位 PDF 结果差的唯一主因或前两大主因
- 给出最小修复并做真实回归，或明确当前缺少的外部条件
