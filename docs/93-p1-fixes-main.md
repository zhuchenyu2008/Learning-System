# 93-P1修复主文档

## 1. 本轮范围
本轮只修 P1 五项：
1. 生成页状态文案与 job 状态联动
2. docx 失败日志定位与链路补强
3. artifact 与笔记分组/展示分离
4. 笔记后异步生成复习卡片
5. 复习时长统计模型修正

## 2. 修复目标
### 2.1 生成页状态文案
- queued / running / completed / failed 要在 UI 中被正确表达
- 不再出现“已创建任务，生成0篇笔记”这种误导提示

### 2.2 docx 链路补强
- 收敛 docx 失败根因
- 增强 docx 提取日志与回归覆盖
- 尽量提升真实复杂 docx 的稳定性

### 2.3 artifact 与笔记分层展示
- summary / mindmap 不再默认混入主笔记库视图
- 前端按 note_type/asset type 做分组或过滤

### 2.4 复习卡片派生任务
- 主笔记生成完成后，可异步派生 review_card_generation 任务
- 卡片与主笔记解耦

### 2.5 复习时长统计修正
- 引入 review session 化思路
- 让 review_watch_seconds 不再长期卡 0

## 3. 执行原则
- P1 仍然按小块拆 subagent
- 先修 UI 状态与 docx，再修分层展示与卡片任务，再修复习时长
- 若现场日志缺失，由主 agent 协调真实环境补位，但不跳过文档与验收

## 4. 验收标准
1. 生成页状态文案与任务状态一致
2. docx 问题要么修复，要么拿到唯一精确根因
3. 主笔记库不再把 summary/mindmap 默认混出
4. 复习卡片生成具备异步派生任务链
5. review 时长统计能稳定增长并落库
