# 00-主任务文档：learning-system

## 1. 项目目标
构建一个**以前后端分离架构实现**的本地文件夹驱动型学习系统，围绕一个用户指定的本地 Markdown/多媒体资料目录工作，完成以下完整闭环：

1. 从录音 / 视频 / 图片 / 文本导入资料。
2. 借助可配置的 OpenAI 兼容 AI、Embedding、语音转文字、多模态 OCR 能力，生成结构化 Markdown 笔记。
3. 将笔记写回本地文件夹，兼容用户继续使用 Obsidian 或其他 Markdown 工具。
4. 基于 FSRS 建立复习计划、复习会话、知识点总结、思维导图生成流程。
5. 提供管理员 / 普通用户登录体系、审计日志、观看时长、复习日志。
6. 支持 Obsidian 同步、数据库导入导出、Docker 部署。

## 3. 成功标准
- 管理员可配置本地工作目录、AI 参数、Obsidian 同步。
- 管理员可导入/扫描录音、视频、图片、文本并生成 Markdown 笔记。
- Markdown 支持内嵌 HTML 与 Mermaid 渲染。
- 系统可基于笔记切片知识点并使用 FSRS 生成复习任务。
- 可手动或定时生成知识点总结与思维导图。
- 普通用户仅可查看内容与写入复习日志，其他写操作禁用。
- 管理员可查看用户登录情况、观看时长、复习情况。
- API 标准化、便于扩展。
- 通过 Docker Compose 一键部署。

## 4. 核心设计原则
- **本地文件夹优先**：Markdown 与原始资料是事实源之一，用户可用任何外部工具操作。
- **数据库负责索引与业务状态**：任务、用户、复习、日志、提炼结果、向量索引元数据在数据库中维护。
- **API 优先**：前后端彻底分离，后端暴露版本化 REST API。
- **异步任务优先**：转写、OCR、总结、思维导图、嵌入向量计算均以任务方式执行。
- **安全默认开启**：角色控制、输入校验、路径约束、敏感配置加密存储。
- **可替换 AI Provider**：兼容 OpenAI 标准；可接入本地模型或第三方兼容服务。

## 6. 用户需求映射
- 需求 1/3：采用前后端分离 + 标准 API。
- 需求 2：系统以本地 Markdown 文件夹作为工作根目录。
- 需求 4：前端采用织物质感 UI 主题。
- 需求 5：支持 Markdown 内嵌 HTML、Mermaid。
- 需求 6/7/12/14/15：AI 配置中心支持 LLM / Embedding / STT / OCR / 提示词。
- 需求 8：支持 Obsidian 同步，当前已定为增强方案优先：本地 Vault 直写 + `obsidian-headless` 集成。
- 需求 9：严格按侧边栏信息架构落地。
- 需求 10：数据库导入/导出。
- 需求 11/17：登录、权限、行为日志、观看时长、复习日志。
- 需求 13：知识点总结/思维导图支持手动与定时。
- 需求 16：复习算法使用 FSRS。
- 需求 18：提供 Docker 部署。

## 7. 默认技术选型（总控定稿）
- 前端：React + Vite + TypeScript + React Router + TanStack Query + Zustand + Tailwind CSS
- 后端：FastAPI + SQLAlchemy + Pydantic + Alembic
- 异步任务：Celery + Redis
- 数据库：PostgreSQL
- 向量：pgvector（优先），不足时预留外部向量库扩展
- 鉴权：JWT Access + Refresh Token
- 定时任务：Celery Beat
- Markdown 渲染：react-markdown + remark-gfm + rehype-raw + rehype-sanitize + Mermaid 组件
- 文件监听：watchdog
- 容器编排：Docker Compose
- PDF 处理：首版直接交给多模态大模型读取/理解，不单独建设复杂版面解析链

## 8. 不做的事（当前阶段）
- 不做移动端 App。
- 不做多人协作文档编辑器。
- 不做复杂 RBAC 自定义角色系统，仅管理员/普通用户两级。
- 不做真正的 Obsidian 插件开发；优先通过文件夹与同步接口兼容。
