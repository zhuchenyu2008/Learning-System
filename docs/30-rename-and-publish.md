# 30-仓库统一改名与发布文档

## 1. 用户要求
将当前项目名称统一改为 `learning-system`，并发布到新建仓库：
- `https://github.com/zhuchenyu2008/Learning-System`

## 2. 本轮目标
1. 将项目目录、文档、README、运行标识、前后端显示名称、Docker/部署相关命名尽量统一到 `learning-system`
2. 保持现有功能与工程化基线不被破坏
3. 初始化/整理 git 仓库并推送到目标 GitHub 仓库

## 3. 命名统一范围
- 项目根目录名
- README 标题与正文中的项目名
- docs 中关键项目名称引用
- 后端应用标题/健康检查返回中的项目名
- 前端页面标题与品牌名
- Docker / compose / 镜像默认命名中可控的项目名
- 其他明显用户可见标识

## 4. 保留边界
- 不做新的功能扩展
- 不借机重构无关模块
- 若存在必须保留的内部技术名，需在回传中说明

## 5. 发布要求
- 若当前目录不是 git 仓库，则初始化 git 仓库
- 关联远程到 `https://github.com/zhuchenyu2008/Learning-System`
- 提交当前代码基线
- 推送到远程默认分支

## 6. 验收标准
- 本地项目名已统一为 `learning-system`
- 关键可见名称已更新
- 代码与构建基线未明显破坏
- 代码已成功推送到目标仓库
