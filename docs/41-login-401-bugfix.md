# 41-登录401缺陷排查文档

## 1. 用户反馈
用户在当前 Docker 全新环境下无法登录，前端提示：`Request failed with status 401`。

## 2. 当前推定范围
优先排查以下链路：
1. 初始管理员是否真实创建成功
2. 登录接口请求体字段与前端提交字段是否一致
3. 密码哈希/校验是否与当前数据库数据一致
4. Docker 全新启动后 entrypoint / seed admin 是否执行成功
5. 前端 api-client / auth-store 是否对登录返回结构解析正确

## 3. 修整目标
- 查清 401 的真实根因
- 修复登录主链
- 在当前 Docker 环境下重新验证登录可用

## 4. 边界
- 不扩展新功能
- 只修登录 401 相关链路
- 修后要给出可用测试账号/验证结果

## 5. 验收标准
- 能明确说明根因
- 登录接口返回成功
- 前端登录可通过同一套账号进入系统
