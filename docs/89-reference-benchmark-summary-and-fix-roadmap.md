当前 `learning-system` 的后测问题，不是零散 bug，而是可归纳为 4 条主线缺口：
1. **主笔记“成品化”边界失守**：检索/抽取/告警等过程性内容泄漏到最终用户笔记正文
2. **产物层与任务层拆分不彻底**：笔记、总结、思维导图、卡片、复习会话未真正解耦
3. **任务状态机没有进入 UI**：queued/running/completed/failed 语义未正确驱动提示与页面状态
4. **复习链只有 FSRS/日志骨架，没有 AI 参与式 review loop**

## 2. 两个参考仓库对当前项目最关键的共同启发
### 2.1 成品笔记原则
- 最终用户笔记必须是“成品”，不能包含检索过程、提示语、TODO、内部元数据、调试说明。
- 检索分数、embedding、调度命令、过程性文本都不应暴露在最终正文。

### 2.2 资产分层原则
- 主笔记、资料、错题、总结、思维导图、复习卡片应有清晰目录与类型边界。
- 复习卡片不写进主笔记正文，应独立落库、独立维护。

### 2.3 任务解耦原则
- 笔记生成、卡片生成、复习会话、统计更新应是不同流程或下游任务，而不是一个大任务硬包。
- 派生任务失败不能污染主笔记成功态。

### 2.4 复习会话原则
- 复习是有开始、作答、评分、讲解、更新、统计的一整条 session 流程。
- 统计与状态回写不是补丁，而是主流程的一部分。

## 3. 对当前项目的修复主轴
### 主轴 A：主笔记成品化
目标：
- 最终 markdown 只保留用户真正需要阅读的笔记内容
- 所有调试/检索/抽取中间态退出主笔记正文

### 主轴 B：产物分层与展示分层
目标：
- 笔记、总结、导图、卡片各走独立目录和独立展示入口
- 单个 artifact 渲染失败不影响主笔记页或其他页面

### 主轴 C：任务状态机与异步链重构
目标：
- 生成页按 job status 表达真实进度
- 卡片生成改为笔记完成后的独立任务

### 主轴 D：复习 session + AI 参与
目标：
- review session 有 start / heartbeat / judge / explain / finish
- AI 给评分建议与讲解，用户可确认或修正
