# 55-第六轮反馈修整文档

## 1. 用户反馈
用户在当前 Docker 环境继续手测后反馈以下问题：
1. 笔记生成页来源资产列表勾选状态显示不稳定：实际已选中但 UI 不显示，需多点几次才显示
2. 需要在来源资产列表里直接删除来源资产
3. 使用 docx / txt / md 作为来源上传后，点击开始生成仍报 `500`
4. 70MB wav 音频上传时报 `413`
5. 用户活动概览里观看时长、页面访问仍为 0

## 2. 目标
对以上 5 个问题做第六轮定向修整，并在修后重新回归验证。

## 3. 拆分策略
- 先修前端来源资产交互问题
- 再修后端上传/生成/活动统计链路
- 最后做统一回归验证与 Docker 重启复测

## 4. 子任务拆分
| SA | 目标 | 类型 |
|---|---|---|
| SA-26A | 来源资产列表交互与删除功能修整 | 实现 |
| SA-26B | 上传/生成/活动统计后端链路排障修复 | 实现 |
| SA-26C | 第六轮问题回归验证 | 验证 |
| SA-26D | 修后 Docker 普通流程重启 | 部署 |

## 5. 验收标准
- 资产勾选状态稳定显示
- 来源资产支持删除
- docx/txt/md 生成笔记链路可用
- 70MB wav 上传链路可用或给出明确受控方案
- 活动概览出现真实非零数据
